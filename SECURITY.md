# Security Policy

## Supported Versions

Security fixes are provided on the latest stable release line.

## Reporting a Vulnerability

Please do not open public issues for sensitive vulnerabilities.

Use private contact through GitHub Security Advisories if available, or open a private channel with the maintainer account `NeuTroNBZh`.

When reporting, include:

- affected version/tag
- reproduction steps
- potential impact
- proposed mitigation if known

## Response Goals

- acknowledgement within 72 hours
- triage and severity assessment
- fix timeline communicated after triage
